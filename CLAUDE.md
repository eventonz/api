# Evento Node API

Node.js rewrite of the Evento API using Fastify framework. Replaces the legacy ColdFusion API with modern, performant architecture.

## Architecture

- **Framework**: Fastify (Node.js)
- **Database**: PostgreSQL (DigitalOcean managed)
- **Cache**: Redis/Valkey (DigitalOcean managed)
- **Auth**: JWT + Bearer tokens with SHA256 hashed API keys
- **Process Manager**: PM2 (cluster mode)
- **Reverse Proxy**: Nginx
- **SSL**: Let's Encrypt (auto-renewing)

## Auto-Deploy Setup

Every push to `main` branch automatically deploys to production via GitHub Actions.

**Workflow**: `.github/workflows/deploy.yml`

**GitHub Secrets Required**:
- `DROPLET_HOST` - Droplet IP address
- `DROPLET_SSH_KEY` - Private SSH key for root access

**What Happens on Push**:
1. GitHub Action triggers
2. SSHs into droplet
3. Pulls latest code (`git pull origin main`)
4. Installs dependencies (`npm install --production`)
5. Restarts PM2 (`pm2 restart evento-api`)

**Manual Deploy** (if needed):
```bash
ssh root@<DROPLET_IP>
cd /root/node-api
git pull origin main
npm install --production
pm2 restart evento-api
```

## SSL Certificate

**Provider**: Let's Encrypt  
**Domains**: eventoapi.com, www.eventoapi.com  
**Expires**: 90 days from issue (auto-renews)  
**Auto-Renewal**: Certbot timer runs twice daily, renews 30 days before expiry

**Certificate Location**: `/etc/letsencrypt/live/eventoapi.com/`

**Manual Renewal** (if needed):


```bash

```

## Endpoints

### Public (No Auth)
- `GET /` - ASCII art homepage
- `GET /health` - Health check (used by load balancer)

### API v1 (Requires Bearer Token)
- `GET /v1/events/:appid` - All events for app
- `GET /v1/events/:appid/upcoming` - Upcoming events
- `GET /v1/events/:appid/past` - Past events
- `GET /v1/config/:raceid` - Race configuration
- `GET /v1/athletes/:raceid` - Athletes/participants
- `GET /v1/tracks/:raceid/:athleteid` - Live tracking data
- `POST /v1/raceresult/:rr_eventid` - RaceResult webhook
- And more... (see `src/routes/v1/` for full list)

## Creating a New Droplet

If you need to spin up a fresh droplet (e.g., staging environment):

1. **Edit droplet name** in `setup-droplet.sh` (line 16):
   ```bash
   DROPLET_NAME="evento-node-api-staging"
   ```

2. **Run setup**:
   ```bash
   ./setup-droplet.sh
   ```

3. **The script will**:
   - Create droplet
   - Install Node.js, PM2, Nginx, Certbot
   - Clone code from GitHub
   - Prompt for environment variables
   - Configure Nginx
   - Set up SSL with Let's Encrypt

4. **Point DNS** to the new droplet IP when prompted

## Switching to Load Balancer (Future)

When ready to add a DigitalOcean Load Balancer:

1. Create load balancer in DO dashboard
2. Add droplet(s) to LB pool
3. Configure SSL on LB (not droplet)
4. Update DNS to point to LB
5. Switch nginx config on droplet:
   ```bash
   scp nginx-loadbalancer.conf root@<DROPLET_IP>:/etc/nginx/sites-available/evento-api
   ssh root@<DROPLET_IP>
   nginx -t && systemctl reload nginx
   certbot delete --cert-name eventoapi.com
   ```

## Performance

- **PM2 cluster mode** - Uses all available CPU cores
- **Redis caching** - 2-minute TTL on event lists (configurable)
- **Nginx keepalive** - Connection pooling to app
- **Database pooling** - Managed by pg module
- **Response times** - Typically <50ms (cached), <200ms (database query)
