# Follow

Manage athlete follows for live tracking. Users can follow specific athletes to receive push notifications and filter tracking views.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | `/v1/follow` | Follow an athlete |
| DELETE | `/v1/follow` | Unfollow an athlete |

## Authentication

Requires Bearer token in Authorization header.

## POST /v1/follow

Add an athlete to the user's follow list.

### Request Body

```json
{
  "event_id": 123,
  "player_id": "device-uuid-abc123",
  "number": "42",
  "contest": 1
}
```

### Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `event_id` | integer | **yes** | Event (race) ID |
| `player_id` | string | **yes** | Device/player identifier (UUID) |
| `number` | string | **yes** | Athlete bib/race number |
| `contest` | integer | no | Contest/race ID (for multi-race events) |

### Response (201 Created)

```json
{
  "response": "success"
}
```

### Response (405 Not Allowed)

Athlete already followed by this player.

```json
{
  "response": "not allowed"
}
```

### Error Responses

| Status | Description |
|--------|-------------|
| 401 | Missing or invalid Bearer token |
| 405 | Already following this athlete (unique constraint violation) |

## DELETE /v1/follow

Remove an athlete from the user's follow list.

### Request Body

```json
{
  "event_id": 123,
  "player_id": "device-uuid-abc123",
  "number": "42"
}
```

### Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `event_id` | integer | **yes** | Event (race) ID |
| `player_id` | string | **yes** | Device/player identifier (UUID) |
| `number` | string | **yes** | Athlete bib/race number |

### Response (200 OK)

```json
{
  "response": "success"
}
```

Note: Returns success even if the athlete wasn't followed (idempotent delete).

## Notes

- `player_id` is a device-specific UUID generated by the app
- The combination of `race_id`, `player_id`, and `raceno` must be unique
- `contest` is optional — used for multi-race events to filter by specific race
- Follows are used to:
  - Send push notifications when followed athletes pass timing points
  - Filter tracking map views to show only followed athletes
  - Highlight followed athletes in results tables
